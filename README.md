# 隧道泵站自动控制系统 V5.7.25_TwinStateHighlightPro

## 本版重点

本版基于 V5.7.24 的原材质保护、底板亮度优化和实时数据同步继续增强三维孪生状态显示。

1. 保留 GLB 原始材质，不直接修改水泵、管道、阀门、电机本体颜色。
2. 新增“状态高亮：开/关”，运行、备用、故障、检修、停止可一眼区分。
3. 运行状态：绿色轮廓、绿色底部光圈、绿色标签。
4. 备用状态：蓝色轮廓、蓝色底部光圈、蓝色标签。
5. 故障状态：红色轮廓、红色底部光圈、红色闪烁标签，优先级最高。
6. 检修状态：橙黄色轮廓、橙黄色底部光圈、检修标签。
7. 停止/未绑定：灰色标签，降低存在感，不抢运行和报警设备。
8. 标签增加状态图标，报警模式只突出故障、检修、屏蔽对象。
9. 底板亮度、亮度、对比度、HDR、阴影地台、轮廓描边等现场微调功能继续保留。

## 建议测试流程

1. 运行 `run_app.bat`。
2. 进入“三维孪生”。
3. 导入 `sample_models/ST01.glb` 或现场模型。
4. 打开“外部 GLB 查看器”。
5. 保持“原材质”模式，确认源材质正常。
6. 保持“状态高亮：开”，观察 P1-P4 运行、备用、故障、检修、停止的区分效果。
7. 切换“只看报警”模式，确认故障/检修/屏蔽对象优先显示。

# 隧道泵站自动控制系统 V5.7.13_GLBFix

本版基于 V5.7.12 修改，重点修复三维孪生 GLB 模型导入和加载问题。

## 本版修复

1. 修复 GLB 加载时报 `[Errno 13] Permission denied` 的问题。
2. 清理三维孪生模块对旧版本目录的依赖，不再将运行模型固定写入 `twin_viewer/models/STxx/model.glb`。
3. 模型保存目录改为可写运行目录：
   - 优先：程序目录 `data/models/STxx/STxx.glb`
   - 失败：`LOCALAPPDATA/TunnelPumpControl/models/STxx/STxx.glb`
4. 每个泵站独立保存模型文件，如 `ST01.glb`、`ST02.glb`，不再统一覆盖成 `model.glb`。
5. 重新导入不同模型时，旧模型自动备份。
6. 增加“模型状态检查”“打开模型目录”“清除模型”功能。
7. GLB 查看器、本地 HTTP 服务、`twin_state.json`、绑定表统一写入安全目录。
8. 增加 `logs/glb_viewer_error.log`，便于现场排查。

## 使用步骤

1. 运行 `run_app.bat`。
2. 进入“三维孪生”页面。
3. 选择当前泵站。
4. 点击“导入模型”，选择 `.glb` 或 `.gltf` 文件。
5. 系统会自动复制到安全目录并保存绑定。
6. 点击“模型状态检查”确认文件路径、对象名称、缺失对象。
7. 点击“外部GLB查看器”查看完整三维模型。

## 推荐模型对象命名

- 水泵：P1、P2、P3、P4
- 母管：PIPE_A、PIPE_B
- 液位计：LT01
- 流量计：FT01
- 压力表：PT01
- 摄像头：CAM01
- 标注锚点：ANNO_P1、ANNO_PIPE_A、ANNO_LT01、ANNO_FT01、ANNO_PT01、ANNO_CAM01

## 说明

三维查看器仍采用本地 HTTP 方式打开，避免浏览器直接读取本地文件的限制。V5.7.13_GLBFix 的重点是稳定路径、权限和模型文件管理；泵站控制、Modbus、报表、视频监控等原功能未做结构性调整。


## V5.7.14_TwinBindFix 补充说明

本版针对修正后的 GLB 模型进行绑定显示修复：模型内 P1/P2/P3/P4、PIPE_A 是父级空节点，实际几何网格在子节点。旧版能扫描到对象，但渲染时只匹配叶子网格名称，导致无法按泵/母管状态联动显示。V5.7.14 已改为“子网格继承父级对象编号”，并增加 ANNO_* 标签点击显示。

测试模型已放入 `sample_models/ST01.glb`。


## V5.7.16_TwinRenderFix 更新说明

本版针对 GLB 导入后显示失真、材质发灰、状态颜色覆盖模型原材质的问题做了专项修复：

1. 离线 WebGL 查看器开始读取 GLB 的 baseColorFactor、baseColorTexture 和 TEXCOORD_0；
2. 支持内嵌 JPG/PNG 贴图读取，优先保留模型原始颜色和贴图；
3. 水泵运行、故障、检修等状态不再直接替换模型材质，改为轻量颜色叠加和脉冲提示；
4. 增加“标准 / 工业美化 / 高质量”渲染模式；
5. 增加“状态标签 / 简洁标签 / 选中详情 / 只看报警”标签模式；
6. twin_state.json 继续每 1 秒刷新，模型标签和状态跟随刷新；
7. GLB 文件本体不写入实时参数，避免模型文件频繁写入损坏或权限冲突。


## V5.7.18_TwinRenderPBRPro

本版重点解决：主程序内部参数正常刷新，但三维孪生仍显示备用、0Hz、0A 的状态不同步问题。

- 主程序每秒自动同步 `twin_state.json`。
- 三维查看器每秒读取实时状态文件。
- 增加同步序号、更新时间和状态超时提示。
- 同时写入新旧状态文件路径，避免读取旧示例数据。
- 保留 V5.7.16 的材质保真和渲染增强逻辑。

推荐测试：启动软件后进入“三维孪生”，导入 `sample_models/ST01.glb`，打开外部 GLB 查看器，观察 P1-P4 的状态、频率、电流是否与“泵站监控”卡片一致。


## V5.7.18_TwinRenderPBRPro 更新说明

## V5.7.21_TwinMaterialRestore 更新说明

本版在 V5.7.18 状态同步与 PBR 材质保真的基础上，继续增强数字孪生 GLB 显示效果，重点补齐：HDR 环境贴图文件、阴影地台、轮廓描边。

本版修复三维查看器材质偏蓝、发灰、半透明感过重的问题。默认使用“原材质”模式，GLB 本体颜色、贴图和透明度优先保持原样；状态不再覆盖模型材质，运行/故障/检修/选中通过标签与轮廓描边表达。HDR 环境默认关闭，地台阴影减弱为中性灰轻量承托。


## V5.7.25_TwinStateHighlightPro 更新说明

本版基于 V5.7.21 原材质保护版继续调整三维孪生显示效果，重点解决测试截图中“接近源材质但对比度太黑、底板死黑、阴影偏重”的问题：

1. 默认背景由深灰蓝调整为中性浅灰，降低画面压暗感；
